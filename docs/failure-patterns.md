# Failure Patterns — CCTV Motion Analyzer

## FP-001: 개인정보 외부 유출 및 규제 위반
- **패턴**: 개발 편의성 또는 외부 API 사용을 위해 CCTV 원본 영상을 YouTube(비공개 포함)나 외부 Cloud API로 전송하는 경우.
- **방지책**: 원본 및 식별 가능 영상은 반드시 로컬 `data/` 내부에서만 처리합니다. 외부 알림 연동 시에도 픽셀 프레임 대신 정형화된 JSON 메타데이터와 텍스트 위주로 알림을 구성하며, 이미지 외부 전송은 사전 승인 및 비식별화(블러) 후에만 고려합니다.

## FP-002: DVR-Scan CLI 파라미터 오용
- **패턴**: `dvr-scan` 실행 시 `-o` 플래그를 사용하여 여러 비디오를 디렉터리로 쪼개려 할 때 실패하는 현상.
- **방지책**: DVR-Scan 1.8.2+ 버전에서 `-o`는 단일 `.avi` 병합 출력용입니다. 여러 세그먼트 클립을 디렉터리로 개별 출력하고 싶다면 반드시 `-d` 혹은 `--output-dir` 플래그를 사용하거나, 현재 시스템처럼 `--scan-only`(`-so`)를 활용해 메타데이터 세그먼트만 파싱한 뒤 pipeline에서 제어해야 합니다.

## FP-003: 라이브러리 API 및 가중치명 추측성 단정
- **패턴**: 공식 문서 확인 없이 YOLO 모델명(예: `yolo26n.pt` 등 존재하지 않는 버전)을 사용하거나, 파라미터 구조를 오인해 런타임 에러가 발생하는 경우.
- **방지책**: Ultralytics YOLO 로딩 시 검증된 `yolo11n.pt`를 기본값으로 사용하고, 버전업 시 공식 문서를 엄격히 검증한 뒤 `config.yaml`에 반영합니다.

## FP-004: 매직 넘버 하드코딩
- **패턴**: 모션 감지 감도(`threshold`), 클래스 필터, ROI 영역, 알림 쿨다운 시간 등을 소스코드에 하드코딩하여 다른 CCTV 영상 환경에서 오탐이 폭증하거나 분석이 누락되는 현상.
- **방지책**: 모든 설정 상수는 `config.yaml`에만 명시하고, 실행 코드는 반드시 `src/config.py` 객체를 주입받아 사용하도록 강제합니다.

## FP-005: 프레임 과다 연산으로 인한 성능 저하
- **패턴**: 모션 구간의 모든 프레임(예: 30fps)을 YOLO 추론에 투입하여 연산 지연 및 GPU/CPU 과부하가 걸리는 현상.
- **방지책**: `config.detect.sample_fps`를 1.0 ~ 2.0으로 튜닝하여, 모션 구간 내에서 초당 1~2개 프레임만 골라 샘플링 추론을 수행합니다.
